## 一、产品概述

### 1.1 产品名称

本地智能归档助手（Local Archive Mate）

### 1.2 产品定位

一款**隐私优先**的 Android 离线信息管理工具。用户通过截图、拍照、录音三种方式输入素材，App 在设备本地完成 OCR、语音转文字和结构化推理，自动提取待办事项、时间节点、联系人、金额等关键信息，一键写入系统日历或存入备忘录。

**核心价值：聊完即存，到点提醒，永不遗忘。**

### 1.3 目标用户

| 用户群体 | 典型场景 |
| :--- | :--- |
| 职场白领 | 整理领导微信指令、会议截图、会议录音 |
| 自由职业/个体户 | 整理客户语音留言、转账记录、交货日期 |
| 隐私敏感型用户 | 律师、财务、医护，数据绝对不上云 |

### 1.4 产品原则

1. **隐私第一**：OCR、语音识别、结构化推理全部本地运行，默认零联网
2. **两步闭环**：从导入到归档，核心操作不超过 3 步
3. **人机协同**：AI 提取 + 人工确认，识别结果可编辑可修正

## 三、功能需求

### 模块1：素材导入

#### 1.1 三种导入方式

| 导入方式 | 操作路径 | 单次上限 | 支持格式 |
| :--- | :--- | :--- | :--- |
| 相册导入 | 首页 → 相册图标 | 20 张 | PNG / JPG / HEIC |
| 相机拍照 | 首页 → 相机图标 | 实时 | - |
| 音频导入 | 首页 → 录音图标 | 30 分钟 | m4a / mp3 / wav / aac |

#### 1.2 导入后直接解析

- 图片导入后，直接送入 **PP-OCRv5** 进行文字识别，**无任何图像预处理环节**（无裁剪、无校正、无增强）
- 音频导入后，直接送入 **Gemma 4 E2B** 进行语音转文字

### 模块2：AI 结构化解析

#### 2.1 处理链路

**截图/拍照链路**：
```
图片 → PP-OCRv5 识别 → Qwen3.5-2B 结构化 → 确认页
```

**音频链路**：
```
录音 → Gemma 4 E2B 转写 → Qwen3.5-2B 结构化 → 确认页
```

#### 2.2 统一输出字段

| 字段名 | 类型 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| `contact` | String | 否 | 对话对象/商户名称 | "张总" |
| `action_items` | Array[String] | 是 | 待办事项列表 | ["准备合同", "发邮件"] |
| `deadline` | DateTime | 否 | 截止时间 | "2026-06-25 18:00" |
| `amount` | Float | 否 | 金额 | 299.00 |
| `address` | String | 否 | 地点 | "A座3楼会议室" |
| `summary` | String | 是 | 15 字以内摘要 | "与客户确认合同" |
| `tags` | Array[String] | 否 | 自动分类标签 | ["工作", "合同"] |
| `source_type` | Enum | 是 | 素材来源 | chat / audio / photo / invoice |

#### 2.3 场景化提取重点

| 素材类型 | 重点提取字段 | 次要字段 |
| :--- | :--- | :--- |
| 聊天截图 | `action_items`, `deadline`, `contact` | `summary`, `tags` |
| 会议录音 | `summary`, `action_items`, `deadline` | `tags` |
| 名片拍照 | `contact`（含姓名/电话/公司/职位） | - |
| 发票/票据 | `amount`, `address`, `deadline` | `tags` |
| 手写便签 | `action_items`, `deadline` | `summary` |

## 参考资料：从零构建 SME2 加速的端侧大模型
接下来我们从源码开始，手把手走一遍开启 SME2 加速的完整端侧大模型部署流程。

## 前置准备
请确保以下环境已就绪：
1. Android NDK：推荐 r27+，配置 `$ANDROID_NDK` 环境变量
2. ADB：电脑可正常连接手机，执行 `adb devices` 能识别设备
3. JDK 17：用于 Gradle 编译 Android APP
4. 测试手机：开启开发者选项 + USB 调试，USB 直连电脑

## Step 1：编译推理引擎
为 Android 编译 MNN 引擎动态库 `.so` 与命令行推理工具；SME2 默认开启，可通过编译参数 `-DMNN_SME2=ON/OFF` 手动控制开关。

```bash
# 1. 进入 MNN Android 编译目录
cd MNN/project/android

# 2. 创建64位编译目录
mkdir build_64 && cd build_64

# 3. 执行编译脚本，开启 SME2、LLM、OpenCL、HQQ量化等全量能力
../build_64.sh "-DMNN_SME2=ON -DMNN_KLEIDIAI=ON -DMNN_LOW_MEMORY=true -DMNN_CPU_WEIGHT_DEQUANT_GEMM=true -DMNN_BUILD_LLM=true -DMNN_SUPPORT_TRANSFORMER_FUSE=true -DMNN_ARM82=true -DMNN_USE_LOGCAT=true -DMNN_OPENCL=true -DLLM_SUPPORT_VISION=true -DMNN_BUILD_OPENCV=true -DMNN_IMGCODECS=true -DLLM_SUPPORT_AUDIO=true -DMNN_BUILD_AUDIO=true -DMNN_BUILD_DIFFUSION=ON -DMNN_SEP_BUILD=OFF -DCMAKE_SHARED_LINKER_FLAGS='-Wl,-z,max-page-size=16384' -DCMAKE_INSTALL_PREFIX=."

# 4. 整理编译产出（必须执行，导出so到lib目录供APP引用）
make install
```

### 编译产出关键文件（build_64/）
- `libMNN.so`：MNN 核心推理引擎库
- `llm_demo`：LLM 命令行推理测试工具
- `llm_bench`：性能跑分基准工具

## Step 2：准备模型
### 方案一：直接下载预转换MNN模型（推荐）
```bash
# 进入模型导出目录
cd MNN/transformers/llm/export

# 安装依赖
pip install modelscope

# 一键下载已量化MNN格式多模态大模型
modelscope download --model MNN/Qwen3-VL-4B-Instruct-MNN --local_dir Qwen3-VL-4B-Instruct-MNN
```

### 方案二：手动转换原始HuggingFace模型（自定义量化场景）
```bash
# 1. 进入导出目录
cd MNN/transformers/llm/export

# 2. 安装python依赖
pip install -r requirements.txt

# 3. 下载原生Qwen3-VL模型
modelscope download Qwen/Qwen3-VL-4B-Instruct --local_dir Qwen3-VL-4B-Instruct

# 4. HQQ量化导出MNN格式
python llmexport.py --path Qwen3-VL-4B-Instruct --dst_path Qwen3-VL-4B-Instruct-MNN --export mnn --hqq
```
> 参数说明：`--export mnn` 指定输出MNN模型；`--hqq` 高精度量化方案，兼顾速度与精度

## Step 3：推送文件到手机，命令行验证推理
### 3.1 推送引擎二进制、动态库、模型到手机
```bash
# 推送推理工具与so库
adb push project/android/build_64/llm_demo /data/local/tmp/
adb push project/android/build_64/llm_bench /data/local/tmp/
adb push project/android/build_64/libMNN.so /data/local/tmp/

# 创建模型存放目录并推送模型
adb shell mkdir -p /data/local/tmp/mnn_models
adb push Qwen3-VL-4B-Instruct-MNN /data/local/tmp/mnn_models/
```

### 3.2 进入手机Shell执行推理
```bash
# 进入手机终端
adb shell

# 赋予工具可执行权限
chmod +x /data/local/tmp/llm_demo /data/local/tmp/llm_bench

# 构造测试prompt
echo "你好" > /data/local/tmp/prompt.txt

# 进入临时目录，指定动态库搜索路径并运行推理
cd /data/local/tmp
export LD_LIBRARY_PATH=/data/local/tmp:$LD_LIBRARY_PATH
./llm_demo /data/local/tmp/mnn_models/Qwen3-VL-4B-Instruct-MNN/config.json /data/local/tmp/prompt.txt
```

> 原理说明：Android 默认仅在系统目录查找 `.so`，`LD_LIBRARY_PATH` 追加自定义目录，让程序加载 `/data/local/tmp` 下的 libMNN.so

### 3.3 校验手机是否硬件支持SME2指令集
新开电脑终端执行日志过滤命令：
```bash
adb logcat -d | grep "device supports"
```
正常输出示例：
```
The device supports: i8sdot:1, fp16:1, i8mm: 1, sve2: 1, sme2: 1
```
- `sme2:1` = CPU硬件支持SME2；
- 硬件支持不代表自动加速：编译时必须开启 `-DMNN_SME2=ON`，引擎才会启用SME2优化路径。
- 该日志为CPU硬件检测结果，不受编译开关控制。

## Step 4：构建完整端侧AI Android应用
命令行验证推理正常后，将MNN引擎集成进Android APP，使用官方MnnLlmChat聊天示例工程：
```bash
# 1. 进入官方LLM聊天Demo工程
cd MNN/apps/Android/MnnLlmChat

# 2. Gradle编译Debug包
./gradlew assembleStandardDebug

# 3. 安装APK到手机
adb install app/build/outputs/apk/standard/debug/app-standard-debug.apk
```

### 使用说明
1. APP编译时自动读取 `project/android/build_64/lib/` 下的 `libMNN.so` 打包进APK，无需手动推送so；
2. 若已在Step3推送模型至 `/data/local/tmp/mnn_models/`，打开MNN Chat后在「我的模型」可直接加载；
3. 也可在APP内置模型市场在线下载其他MNN量化模型。
